## Installation

Pick the method that fits your workflow. All three result in a `CLAUDE.md` file in your project root.

<Tabs>
  <Tab title="Universal">
    The fastest way to get started. Run this from your project root:

    <Steps>
      <Step title="Navigate to your project root">
        ```bash
        cd your-project/
        ```
      </Step>
      <Step title="Download CLAUDE.md">
        ```bash
        curl -o CLAUDE.md https://raw.githubusercontent.com/drona23/claude-token-efficient/main/CLAUDE.md
        ```
      </Step>
      <Step title="Start Claude Code">
        Open Claude Code in your project. The file is picked up automatically — no restart or configuration needed.
      </Step>
    </Steps>
  </Tab>
  <Tab title="Clone">
    Use this option if you want to browse and pick a specific profile for your project type.

    <Steps>
      <Step title="Clone the repository">
        ```bash
        git clone https://github.com/drona23/claude-token-efficient
        ```
      </Step>
      <Step title="Pick a profile">
        Choose the profile that matches your workflow:

        | Profile | Best for |
        |---------|----------|
        | `CLAUDE.md` | Universal — works for any project |
        | `profiles/CLAUDE.benchmark.md` | Token-to-green coding benchmarks |
        | `profiles/CLAUDE.coding.md` | Dev projects, code review, debugging |
        | `profiles/CLAUDE.agents.md` | Automation pipelines, multi-agent systems |
        | `profiles/CLAUDE.analysis.md` | Data analysis, research, reporting |
      </Step>
      <Step title="Copy your chosen profile into your project root">
        ```bash
        cp claude-token-efficient/profiles/CLAUDE.coding.md your-project/CLAUDE.md
        ```

        Or copy the base file for the universal default:

        ```bash
        cp claude-token-efficient/CLAUDE.md your-project/CLAUDE.md
        ```
      </Step>
      <Step title="Start Claude Code">
        Open Claude Code in your project. The file is picked up automatically.
      </Step>
    </Steps>
  </Tab>
  <Tab title="Manual">
    Copy the file contents by hand — useful if you want to review or customize before saving.

    <Steps>
      <Step title="Open the source file">
        Navigate to the repository and open `CLAUDE.md`:

        ```
        https://github.com/drona23/claude-token-efficient/blob/main/CLAUDE.md
        ```
      </Step>
      <Step title="Copy the contents">
        Copy the full file contents.
      </Step>
      <Step title="Create CLAUDE.md in your project root">
        Create a new file named `CLAUDE.md` in the root of your project and paste the contents.

        ```
        your-project/
        └── CLAUDE.md
        ```
      </Step>
      <Step title="Start Claude Code">
        Open Claude Code in your project. The file is picked up automatically.
      </Step>
    </Steps>
  </Tab>
</Tabs>

## Verify it's working

Ask Claude to review a simple piece of code. Compare its response against what you'd normally expect.

**Without CLAUDE.md**, a code review prompt returns something like:

```text
Great question! I can see you're working with a loop. Let me take a look at that.
That's a thoughtful piece of code! However, I notice there's actually a subtle bug
here that's worth pointing out. The condition i <= arr.length should be i < arr.length.
Using <= means the loop will run one extra iteration...

Great catch on reviewing this - it's always good practice to double-check loop
boundaries! I hope this review was helpful. Feel free to share more code anytime!
```

**With CLAUDE.md**, the same prompt returns:

```text
Bug: <= causes an off-by-one error. On the final iteration arr[arr.length] is undefined.

Fix:
for (let i = 0; i < arr.length; i++)
```

If your responses are direct and free of preamble and closing fluff, the file is active and working.
