title: 'Versioned configs overview'
description: 'Pre-tuned multi-file configuration sets for different optimization strategies. Choose the version that matches your workflow.'
---

Versioned configuration sets are complete, drop-in CLAUDE.md configurations that go beyond the single-file profiles. Each version ships as a directory of files — an entry-point `CLAUDE.md` plus subdirectories for rules, agent definitions, and reference patterns — so you get a structured, multi-file setup out of the box.

Unlike the single-file profiles (`CLAUDE.coding.md`, `CLAUDE.agents.md`, etc.) which tune tone and output style, the versioned configs also define **tool budgets**, **agent protocols**, and **workflow rules** that govern how Claude executes tasks end-to-end.

## Comparison

| Version | Strategy | Tool Budget | Best For |
|---------|----------|-------------|----------|
| `J-drona23-v5` | Multi-file structured | 50 calls | Complex projects needing detailed workflow rules and agent definitions |
| `K-drona23-v6` | One-shot execution | 30 calls | Tasks that should complete in a single pass with minimal iteration |
| `M-drona23-v8` | Ultra-lean minimum-turn | 20 calls | Cost-sensitive pipelines where every tool call counts |

## How to choose

- Start with **v5** if you need structured multi-step workflows with clear agent protocols.
- Use **v6** if you want faster execution with strict "done means done" rules (no polishing passing code).
- Use **v8** only if you need maximum cost efficiency and your tasks are simple enough for 20 tool calls.

## How versioned configs differ from single-file profiles

| | Single-file profiles | Versioned configs |
|--|---------------------|-------------------|
| File count | 1 file | Multiple files across subdirectories |
| Scope | Tone, output format | Tone + tool budget + agent roles + workflow rules |
| Setup | Copy one file | Copy entire directory |
| Best for | General-purpose tuning | Structured execution strategies |

<Note>
The versioned configs use Claude's multi-file CLAUDE.md composition. When you copy a versioned directory into your project, Claude reads the top-level `CLAUDE.md` entry point, then follows its instructions to load additional files from the subdirectories before starting work.
</Note>
