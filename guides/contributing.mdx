Found something Claude does by default that wastes tokens or gets in the way? If a rule in `CLAUDE.md` could fix it, open an issue and share it with the community.

## How to Submit a Fix

<Steps>
  <Step title="Identify the annoying behavior">
    Describe exactly what Claude does by default that you want to change. Be specific. "Claude is too verbose" is too vague. "Claude opens every code review with a compliment and closes with an offer to help further" is actionable.
  </Step>

  <Step title="Share the prompt that triggers it">
    Paste the prompt or task description that reliably produces the behavior. This lets maintainers reproduce it and verify that a rule fixes it.
  </Step>

  <Step title="Propose the fix rule">
    Write the rule you added to your own `CLAUDE.md` that solved the problem. One or two lines is ideal. If the fix requires more than that, explain why.

    Example submission:

    ```
    Behavior: Claude silently catches exceptions in pipeline steps and
    continues as if nothing failed.

    Prompt: "Process all files in /data and write results to output.json"

    Fix rule: When a step fails, stop immediately and report the full
    error with traceback before attempting any fix.
    ```
  </Step>
</Steps>

Open your issue here: [github.com/drona23/claude-token-efficient/issues](https://github.com/drona23/claude-token-efficient/issues)

<Note>
Community submissions become part of the next version with full credit. Your name and the source issue will be listed in the project's references section.
</Note>
